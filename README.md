  # cic.newassumptions.newvarianceestimator: Changes-in-Changes Estimator

  `cic.newassumptions.newvarianceestimator` is an R package for the Changes-in-Changes estimator and asymptotic inference for empirical quantile-based estimators. It computes the plug-in estimate, provides several confidence interval methods, and includes a diagnostic helper to check whether the input data look compatible with the model assumptions.

  ## Features

  - Point estimation of the CiC parameter from `Y`, `X`, and `Z`
  - Confidence intervals with five methods:
    - `"no-split"`
    - `"split"`
    - `"kde"`
    - `"bse"`
    - `"bpc"`
  - Diagnostic checks via `check_cic_assumptions()`
  - Rcpp-backed computation for the core routines, with a pure R fallback where available
  - Simulation helpers `sim_dgp()`, `qY_dgp()`, and `theta_true()`

  ## Installation

  ```r
  devtools::install_github("Timotheebacchi/cic_package")
  ```

  ## Quick Start

  ```r
  library(cic.newassumptions.newvarianceestimator)

  set.seed(42)
  d <- sim_dgp(2000)

  diag <- check_cic_assumptions(d$Y, d$X, d$Z)
  diag$pass_all

  set.seed(42)
  d_fail <- sim_dgp(500)
  diag_fail <- check_cic_assumptions(d_fail$Y, d_fail$X, d_fail$Z)
  diag_fail$pass_all
  diag_fail$messages

  fit <- cic(d$Y, d$X, d$Z, method = c("no-split", "split", "kde", "bse", "bpc"))

  fit
  fit$ci
  summary(fit)
  ```

  ## Diagnostics

  `check_cic_assumptions()` is a screening step before estimation. It looks at
  the sample sizes, the i.i.d. and continuity checks, the estimated tail
  behavior of $Y$, the estimated boundary behavior of $U = F_Z(X)$, and the
  bandwidth choice. It is meant to flag samples that are fragile for the CiC
  asymptotic theory, not to prove that the assumptions are mathematically true.

  The function returns a list with:

  - `pass_all`: overall logical result
  - `metrics`: sample ratios, tail indices, boundary estimates, and related checks
  - `messages`: warnings or success messages

  How to read it:

  - `pass_all = TRUE` means the sample did not trigger any hard failure in the
    diagnostic rules.
  - `pass_all = FALSE` means at least one check crossed a threshold that the
    package treats as a serious warning.
  - `metrics` is where you inspect the actual estimated values if you want to
    understand why the sample passed or failed.
  - `messages` explains the result in plain language and should be read first
    when the flag is not what you expected.

  `pass_all` is a summary flag, not a proof that the CiC assumptions hold. It
  can flip to `FALSE` when one estimated component looks fragile in finite
  samples, especially the boundary-density check used in the quick start. If
  that happens, inspect `diag$messages` and `diag$metrics` rather than treating
  the flag as a hard theorem.

  The sample generated by `set.seed(42); sim_dgp(500)` is a concrete example
  where the diagnostic can fail in this implementation. That does not mean the
  CiC model is rejected in theory; it means the finite-sample diagnostic finds
  the boundary/tail behavior too unstable to give a clean pass.

  ## Assumptions Reminder

  The package is built for the CiC setup described in the manuscript. Before
  interpreting the output, the input data should be checked against the main
  assumptions used by the estimator:

  - the observations should look approximately i.i.d. and continuous enough for
    the rank transformation to make sense;
  - the empirical quantile function of $Y$ should stay below a boundary envelope
    of the form $C_Y t^{-d_1}(1-t)^{-d_2}$ on the interior of $(0,1)$;
  - the transformed covariate ranks $U = F_Z(X)$ should admit a smooth density
    that can be screened against $C_U u^{-b_1}(1-u)^{-b_2}$;
  - the outcome distribution should have tails that are not too heavy;
  - the combined tail and boundary behavior should stay within the rate
    conditions required by the asymptotic theory;
  - the smoothing bandwidth should be reasonable for the sample size.

  The diagnostic helper `check_cic_assumptions()` is a screening tool for these
  conditions. It evaluates the sample-based quantile and density estimates on
  interior grids, then turns suspicious shape mismatches into warnings. It is
  intentionally conservative and should be read as a warning system, not as a
  formal proof that the model is valid.

  In practice, `cic()` can receive one method or several methods at once, and it
  returns the intervals in the requested order. The asymptotic methods are
  `no-split`, `split`, and `kde`, while `bse` and `bpc` are bootstrap
  comparisons that help assess robustness in small samples.
